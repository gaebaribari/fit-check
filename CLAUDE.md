# I want
나는 틱톡 마케터 대행사에 다니는 마케터고
빠르게 이번 캠페인과 핏이 맞는 크리에이터를 찾는 서비스를 만드려고 해 

그래서 모아놓은 틱톡주소들을 쫙 복붙하면
아래에 프사와 틱톡아이디, 닉네임, 팔로워, 최근게시물 10개 썸네일이 나와
그리고 왼쪽에 있는 체크표시로 빠르게 선택한후 내보내기 하기 

그리고 팔로워 100명 이하 1000명이하는 보이지않게 할수도있고
같은 캠페인에서 한번 지나쳤던 크리에이터는 히스토리에서관리되고있으니까 보이지않게 하고

# 설계 (v1 — 크리에이터 찾기 기능 집중)

캠페인별 관리는 나중에. 우선 **크리에이터 조회 + 필터 + 선택 + 내보내기**에 집중.

## 기술 스택
- **Frontend**: Next.js (App Router) + TypeScript + Tailwind CSS
- **Backend**: Next.js API Routes
- **DB**: MySQL
- **틱톡 데이터**: TikTok 비공식 스크래핑 (RapidAPI 또는 직접 파싱)
- **내보내기**: CSV 다운로드

## API 구조

| 엔드포인트 | 설명 |
|---|---|
| `POST /api/tiktok/lookup` | 틱톡 URL 목록 → 프로필+최근영상 정보 반환 |
| `GET /api/skipped` | 스킵된 크리에이터 목록 |
| `POST /api/skipped` | 크리에이터 스킵 처리 (배열) |
| `DELETE /api/skipped/[username]` | 스킵 해제 |
| `POST /api/export` | 선택한 크리에이터 CSV 내보내기 |

## 동작 흐름

1. 틱톡 URL 여러 개를 textarea에 붙여넣기
2. "조회하기" 클릭 → `POST /api/tiktok/lookup`으로 일괄 조회
3. 결과 카드 리스트 표시 (프사, 아이디, 닉네임, 팔로워, 최근게시일, 썸네일 10개)
4. 필터 적용 — 팔로워 기준 미달 숨김, N일 내 게시물 없는 계정 숨김, 스킵 히스토리 숨김(디폴트: 숨김 / 토글로 보고 닫을수있음)
5. 체크박스로 원하는 크리에이터 선택
6. "내보내기" → CSV 다운로드 / "스킵처리" → 선택 안한 크리에이터 DB에 저장
